1 · Scope

The current TUI loses context on view switches, breaks below ~100 cols, and composites modals by reflowing content around them (#19). This redesign rebuilds the shell on a view stack with one overlay compositor, one responsive algorithm, and a GitHub-parity PR review view.

In: dashboard shell · compositor · command layers · wizard keys · full-screen config · PR review view.
Out: comment authoring (stretch) · worktree-diff base-cycling (folds in behind the PR view later) · : command palette.

Figures below are normative mockups, not illustrations: layout, glyph choice, spacing, and color roles (Catppuccin Mocha tokens per docs/grove-tui-context.md) are the expectation. Copy text is sample data.

Amended 2026-07-23: UX research review R1–R6 adopted (goto layer · last-worktree toggle · filterable help · h/l aliases · naming pass · PR-browser keys); R7–R9 tracked in §10. UI addenda: D14 focus tint · D15 width-floor guard · help-match highlighting (D12) · ↩ last-worktree marker (D5). Implementation deltas mapped in §11; full-screen state figures in Appendix B. 2 · Decisions (locked)
D1
architecture
List-first (1a). ≥110 cols: docked detail pane (3a), tab focuses it. Below: 1-line strip + tab-sheet. The list never sheds for chrome.
D2
compositor
One overlay layer, uniform dim. Overlays never reflow background content (1g). Fixes #19.
D3
row spec
Worktree name is the identity column; 11ch right-aligned counts (↑n ↓nnn ~nn); 3ch age, no "ago"; glyph tail ⬢⬡◆ (4b).
D4
secondary col
Config list_secondary = branch | dir | off. Default branch, rendered as dim ⎇ chip only when ≠ name (prefix-dupes count as equal). Directory always in detail → Git.
D5
navigation
Vim only: j/k · ctrl+d/u · g g / G top/bottom (bare g opens the goto layer, R1) · h/l close/open detail · - toggles last worktree (R2; its target row carries a dim ↩). Number gutter and 1-9 quick-switch removed. Position n/52 in header.
D6
truncation
Middle-truncate; ticket-id prefix and last segment always survive. Name floor 16ch; the ⎇ chip sheds entirely before the name drops below 24ch.
D7
wizard
⏎ next · shift+tab back · esc cancels from any step and returns to the origin view with cursor restored (1e).
D8
config
Full-screen editor, not a modal (1f). Edit-in-place, dirty-field tracking, save/discard prompt on esc.
D9
upgrade
Header badge ⭡x.y.z (the available version); u opens a confirm overlay, never fires blind; listed in help.
D10
PR review
GitHub-parity tabs in this iteration (6a–6d). All keys view-local.
D11
layers
Prefix keys + which-key strip (5b). Strip is docked chrome above the footer — viewport shrinks, selection stays visible, input is never gated: b s executes instantly. Within g, g = top — gg stays an instant chain (R1).
D12
help
? overlay filters as you type across key, label, description with matched substrings highlighted; entries carry plain synonyms — fork (copy) (R3). Fig B1.
D13
naming
Layer key = first letter of its label; labels are verb + object. b p pull from worktree (was b y) · w d delete multiple (was w a) · b s switch branch here (R5).
D14
focus
tab / l focuses the detail pane (≥110 cols): border tints Primary (DetailBorder focus variant), list cursor dims, j/k scroll the pane; h/esc return. Fig B2.
D15
floor guard
Below 44 cols a one-line guard replaces the UI ("grove needs ≥44 columns"); state preserved, restored instantly on resize, never crashes. Fig B3.

4 · Compositor & overlays

Single z-layer owned by the root model. Help, wizard, confirms, upgrade render through it: background dims uniformly, is never re-laid-out, and keeps its scroll. esc pops exactly one level. The which-key strip is NOT an overlay — it is docked chrome (D11). Config replaces the screen (D8).

7 · PR review view

Pushed onto the stack from a PR row (⏎) or worktree (g d); n on a PR row opens the New Worktree wizard for that PR (Fig 3). Data: gh pr view --json + gh pr diff, cached per PR. Tabs: [1] conversation · [2] commits · [3] checks · [4] files.

Folder tree rail (4d rollups, D6 path compression) + hunk pane. t toggles rail; auto-hidden <100 cols. v marks viewed (dim ✓, persisted per PR across restarts). ⏎ opens $EDITOR at file:line. z folds runs of removed lines.
commits (6c)
⏎ scopes the files tab to one commit (header shows sha); a resets to all.
conversation (6b)
Read-only timeline: comments, review threads, check states. ⏎ on a thread jumps to its file:line in tab 4; B opens browser to reply.

9 · Acceptance criteria
✓ 80×24 renders with zero wrapped rows; every width ≥44 cols keeps one worktree per row
✓ esc from any view or overlay returns to the exact origin — cursor and scroll restored
✓ overlays dim the background without reflowing it (config screenshot bug #19 gone)
✓ b s completes with no strip-induced latency; strip visible within one frame of the prefix
✓ middle-truncation never drops a ticket id (gal-1349-…, GAL-1656, mj-1475-…)
✓ viewed-file state per PR survives restart
✓ ⏎ in a hunk lands $EDITOR at the exact file:line; quitting restores the view unchanged
✓ config esc with dirty fields always prompts; upgrade never runs without confirm
✓ filter renumbers nothing (no gutter) and header position n/total tracks the filtered set
✓ - returns to the previously active worktree (dim ↩ marks it); a second - returns back
✓ ? help filters as you type across key, label, and description — matches highlighted
✓ focused pane: Primary border + pane-scoped footer; h/esc return with list cursor unchanged
✓ below 44 cols only the guard renders; widening restores the exact prior state
