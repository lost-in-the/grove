grove — TUI UX research review
addendum to the delivery doc · benchmarked against lazygit, helix, ranger/yazi, Charm guidance, keyboard-ergonomics literature, and word-finding (anomic-aphasia) review criteria
1 · What the redesign already gets right
✓ Which-key layers (D11) — matches helix's minor-mode pattern and solves the discoverability gap lazygit users file issues about. Recognition over recall, no memorization tax.
✓ Detail on demand (D1) — lazygit's core praise is "shows the data you need when you need it without overwhelming"; list-first + tab-to-detail is that shape.
✓ One compositor, uniform dim (D2) — overlays that never reflow background are the established modal pattern; fixes #19 the right way.
✓ Defined minimum width (44-col floor) — TUI guidance: pick a floor, degrade predictably, never crash on resize.
✓ Keyboard-complete — the repo's no-mouse constraint is honored in every flow.
2 · Adopt — R1–R6
R1
g is double-booked — spec conflict. D5 binds g/G to top/bottom; §5 also defines a "g go" prefix layer. Both can't hold. Fix per helix's goto minor mode: g opens the layer, g g = top, G stays global bottom. Muscle memory (gg) survives; the strip renders after the first g. Figure below.
R2
`-` switches to the previous worktree. Grove's own pitch is the interrupt ("a high-priority PR lands") — but the return leg of that journey has no key. `git switch -`, tmux last-window, and vim ctrl-^ all reserve one keystroke for "back to what I was doing". Cheapest high-value key in this review.
R3
Filterable help. ? opens help; typing filters by key, name, or description (same / muscle as the list). This is lazygit's most-requested discoverability fix, and it is the anomic-aphasia mitigation: retrieval by any remembered word, not the exact term. Help descriptions carry synonyms — "fork (copy, duplicate)".
R4
h/l spatial aliases. l opens detail (deeper), h closes/backs out — the ranger/yazi/lf grammar vim hands already know. Puts the #2 action (peek detail) on home row; h/l are unbound today, so zero cost. tab/esc remain.
R5
Naming pass for word-finding — table in §3. Rule set: layer key = first letter of its visible label; strip label = verb + object; help adds a plain-language synonym.
R6
PR/issue browser keys are unspecified. §5 covers the review view, not the g p browser itself. Fig 3 implies n (new worktree from PR); ⏎ opens review. Add the row to §5 and one acceptance criterion.

4 · Consider / verify — R7–R9
R7
Stale-first cleanup. The cleanup journey is find-then-delete; sort cycles name → age → dirty but stale (✗) has no fast path. Option: add stale to the o cycle, or a /is:stale filter token. Small, not urgent.
R8
u on a prime key for a rare action. Ergonomics literature maps frequency to reachability; upgrade is rare and u is prime home-adjacent real estate (and vim-undo muscle memory). D9's confirm makes it safe as-is; parking it in the goto layer later costs nothing.
R9
Terminal-protocol check. lazygit documents that bindings beyond plain letters and ctrl+letter are unreliable in Terminal.app and MinTTY, and tmux strips extended keys unless configured — grove users live inside tmux. shift+tab (backtab, CSI Z) is old and broadly safe, but add a golden/manual matrix row: shift+tab and ctrl+d/u under tmux, Terminal.app, MinTTY.
